ลองอธิบายแบบเปรียบเทียบกับ Cisco Router นะครับ:

1. Running = Running-config ใน Cisco
- คือการตั้งค่าที่ทำงานอยู่จริงๆ ตอนนี้
- เหมือนกับที่เราพิมพ์ show running-config ใน Cisco

2. Startup = Startup-config ใน Cisco
- คือการตั้งค่าที่จะถูกใช้ตอนเปิดเครื่องใหม่
- เหมือนกับที่เราพิมพ์ show startup-config
- ถ้าจะให้ running-config ไปอยู่ใน startup-config ต้องใช้คำสั่ง copy running-config startup-config

3. Candidate = คล้ายๆ กับการทำ "configure terminal" ใน Cisco แต่ดีกว่า
- ใน Cisco เวลาเราพิมพ์ configure terminal แล้วตั้งค่าอะไร มันจะเปลี่ยน running-config ทันที
- แต่ candidate เป็นเหมือน "สนามทดลอง" ที่เราสามารถลองตั้งค่าต่างๆ ได้ โดยยังไม่กระทบกับระบบจริง
- เปรียบเหมือนเรามี notepad ที่เราสามารถร่างการตั้งค่าทั้งหมดได้ก่อน
- เมื่อเราพอใจกับการตั้งค่าทั้งหมดแล้ว ค่อยสั่งให้นำไปใช้จริงใน running

ตัวอย่างการใช้งาน Candidate:
1. สมมติเราต้องตั้งค่า VLAN, IP, Routing หลายๆ อย่างพร้อมกัน
2. แทนที่จะตั้งทีละคำสั่งแล้วมีผลทันที เราสามารถร่างการตั้งค่าทั้งหมดใน candidate ก่อน
3. ตรวจสอบความถูกต้องของการตั้งค่าทั้งหมด
4. ถ้าพอใจแล้วค่อยสั่งให้นำไปใช้ใน running ทั้งชุด
5. ถ้าไม่พอใจก็สามารถทิ้งการตั้งค่าใน candidate ได้ โดยไม่กระทบระบบที่ทำงานอยู่

ข้อดีคือ:
- ปลอดภัยกว่า เพราะได้ตรวจสอบการตั้งค่าทั้งชุดก่อนนำไปใช้จริง
- ลดความเสี่ยงจากการตั้งค่าผิดพลาดระหว่างทาง
- สามารถเตรียมการตั้งค่าที่ซับซ้อนได้ โดยไม่กระทบระบบที่ทำงานอยู่
